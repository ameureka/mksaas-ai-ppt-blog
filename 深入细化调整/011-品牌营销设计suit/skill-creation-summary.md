## 🎓 关键学习点

### 1. Skill 提炼方法

**识别可复用性**:
- ✅ 流程是否标准化?
- ✅ 输出是否可模板化?
- ✅ 工具是否可自动化?

**提炼步骤**:
1. 回顾原始任务执行过程
2. 识别可复用的核心要素
3. 设计标准化的执行流程
4. 创建文档模板和脚本
5. 编写 SKILL.md 和 YAML
6. 测试和优化

### 2. Description 设计技巧

**✅ 好的 Description**:
```yaml
description: 为网站或产品设计完整的品牌战略方案,包括品牌定位、目标受众分析、视觉识别系统(Logo/配色/字体)、内容营销策略、用户增长策略和商业模式设计。适用于需要品牌重塑、新产品发布或营销策略规划的场景。
```

**包含要素**:
- ✅ 做什么: 设计品牌战略方案
- ✅ 包含什么: 品牌定位、视觉系统、营销策略...
- ✅ 何时使用: 品牌重塑、新产品发布、营销策略规划
- ✅ 关键词: Logo、配色、字体、营销、增长

---

### 3. 指令编写技巧

**清晰的结构**:
```markdown
## 阶段 N: 阶段名称 (英文名)

### 目标
明确说明这个阶段要达成什么

### 执行步骤
1. 步骤 1
   - 子步骤 a
   - 子步骤 b
2. 步骤 2
   ...

### 输出
- 输出文件 1
- 输出文件 2
```

**关键原则**:
- 分步骤指导
- 明确输入输出
- 边界条件说明
- 错误处理指南

---

### 4. 脚本设计技巧

**防错设计**:
```bash
# 检查依赖
command -v convert >/dev/null 2>&1 || {
    echo "错误：需要安装 ImageMagick"
    exit 1
}

# 检查输入
if [ ! -f "$INPUT_FILE" ]; then
    echo "错误：文件不存在"
    exit 1
fi

# 安全设置
set -e  # 遇到错误立即退出
```

**幂等性**:
- 重复执行不产生副作用
- 覆盖旧文件而非追加
- 提供备份机制
