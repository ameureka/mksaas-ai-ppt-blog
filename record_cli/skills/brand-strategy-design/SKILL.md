# Brand Strategy Design Skill

## 概述

这个技能帮助你从零开始设计完整的品牌战略方案,通过结构化的提问和分析流程,生成包含品牌定位、视觉系统、营销策略和执行路线图的专业文档。

**核心价值**:
- 系统化的品牌战略设计流程
- 从战略到执行的完整方案
- 可直接落地的技术实施指南
- 标准化的文档和配置输出

**适用场景**:
- 网站/产品品牌重塑
- 新产品品牌设计
- 营销策略规划
- 品牌统一项目

## 执行流程

### 阶段 1: 需求分析 (Discovery)

**目标**: 深入理解产品定位和目标受众

**执行步骤**:

1. **产品定位分析**
   - 询问产品的核心价值主张
   - 确认产品类型(B2B/B2C/工具/内容站等)
   - 识别主要功能和差异化优势
   
2. **目标受众画像**
   - 确定主要用户群体(年龄/职业/需求)
   - 分析用户痛点和核心诉求
   - 绘制用户旅程地图
   
3. **竞争分析**
   - 识别主要竞品
   - 分析差异化竞争点
   - 确定市场定位

4. **品牌调性确认**
   - 专业/创新/活力/科技等风格选择
   - 确认品牌关键词(3-5个)
   - 定义品牌承诺(一句话)

**输出**: 需求分析报告(Markdown)

---

### 阶段 2: 品牌战略设计 (Strategy)

**目标**: 设计完整的品牌战略框架

**执行步骤**:

1. **品牌定位画布**
   ```
   目标用户 + 核心痛点 + 解决方案 + 差异化优势
   ```
   
2. **品牌命名建议**
   - 提供 3-5 个品牌名称方案
   - 分析每个名称的优缺点
   - 推荐最佳方案
   
3. **品牌标语(Slogan)**
   - 设计 3 个品牌标语选项
   - 体现核心价值和差异化
   
4. **品牌故事**
   - 撰写品牌起源故事
   - 定义品牌使命和愿景

**输出**: 品牌战略文档(Markdown)

---

### 阶段 3: 视觉识别系统 (Visual Identity)

**目标**: 设计完整的视觉识别系统

**执行步骤**:

1. **Logo 设计方案**
   - 提供 3 个 Logo 设计方向
   - 每个方案包含:
     - 设计理念说明
     - 视觉描述(ASCII 艺术)
     - AI 生成 Prompt(Midjourney/DALL-E)
     - 优缺点分析
   - 推荐最佳方案
   
2. **配色系统**
   - 主色调(Primary Color)
   - 辅助色(Secondary Color)
   - 渐变色(Gradient)
   - 中性色(Neutral Colors)
   - 深浅模式适配
   
3. **字体系统**
   - 中文字体推荐
   - 英文字体推荐
   - 字体使用规范
   
4. **视觉规范**
   - Logo 使用规范
   - 最小尺寸和安全区
   - 禁用示例

**输出**: 
- Logo 设计方案文档(Markdown)
- 品牌配置文件(JSON)
- 视觉规范手册(Markdown)

---

### 阶段 4: 内容营销策略 (Content Marketing)

**目标**: 设计内容营销和 SEO 策略

**执行步骤**:

1. **内容策略**
   - 博客内容重新定位
   - 内容分类和标签系统
   - 内容更新节奏
   
2. **SEO 关键词策略**
   - 主关键词(高搜索量)
   - 长尾关键词(高转化率)
   - 地域关键词(本地化)
   
3. **社交媒体矩阵**
   - 平台选择(小红书/抖音/知乎等)
   - 内容形式(图文/视频/直播)
   - 发布频率和时间
   
4. **内容分发策略**
   - 核心内容 → 多平台改编
   - 内容复用和再创作

**输出**: 内容营销策略文档(Markdown)

---

### 阶段 5: 用户增长策略 (Growth Strategy)

**目标**: 设计用户获取和留存策略

**执行步骤**:

1. **增长漏斗设计**
   - 获客(Acquisition)
   - 激活(Activation)
   - 留存(Retention)
   - 变现(Revenue)
   - 推荐(Referral)
   
2. **北极星指标**
   - 定义核心增长指标
   - 设计关键指标树
   
3. **冷启动策略**
   - 前 3 个月执行计划
   - 每月目标和行动项
   
4. **增长实验**
   - A/B 测试计划
   - 优化迭代策略

**输出**: 用户增长策略文档(Markdown)

---

### 阶段 6: 商业模式设计 (Business Model)

**目标**: 设计可持续的商业模式

**执行步骤**:

1. **当前阶段模式**
   - 收入来源
   - 成本结构
   - 盈亏平衡点
   
2. **未来扩展模式**
   - 会员订阅
   - 软件售卖
   - 企业定制
   
3. **开源策略**(可选)
   - 开源内容范围
   - 商业化路径

**输出**: 商业模式文档(Markdown)

---

### 阶段 7: 执行路线图 (Execution Roadmap)

**目标**: 制定可执行的实施计划

**执行步骤**:

1. **分阶段执行计划**
   - 第 1-2 周: 品牌视觉设计
   - 第 3-4 周: 内容优化
   - 第 5-8 周: 流量获取
   - 第 9-12 周: 数据驱动优化
   
2. **关键决策点**
   - 品牌名称确认
   - Logo 风格选择
   - 配色方案确认
   - 域名注册
   
3. **风险控制**
   - 主要风险识别
   - 应急预案
   - 回滚机制
   
4. **成功指标(KPI)**
   - 第一季度目标
   - 第二季度目标
   - 量化改善指标

**输出**: 执行路线图文档(Markdown)

---

### 阶段 8: 技术实施工具 (Technical Implementation)

**目标**: 生成技术实施所需的脚本和配置

**执行步骤**:

1. **品牌配置文件**
   - 生成 `brand-config.json`
   - 包含品牌名称、配色、字体、SEO 配置
   - 替换规则定义
   
2. **Logo 生成工具**
   - AI 生成 Prompt 文档
   - 工具推荐和使用指南
   - 外包平台对比
   
3. **Favicon 生成脚本**
   - 自动化 Favicon 生成脚本
   - 支持多平台(iOS/Android/Web)
   - 自动压缩优化
   
4. **品牌资产目录**
   - 创建标准化目录结构
   - 使用规范文档
   - 文件命名规范

**输出**:
- `brand-config.json` - 品牌配置
- `generate-favicon.sh` - Favicon 生成脚本
- `brand-assets/` - 资产目录结构
- 工具使用文档

---

### 阶段 9: 快速开始指南 (Quick Start)

**目标**: 提供立即可执行的行动指南

**执行步骤**:

1. **3 个执行方案**
   - 快速启动(1 小时,低成本)

   - 专业外包(3-7 天,高质量)

   - 必须决策事项

   - 可选决策事项
   
3. **工具准备**
   - 必需工具列表
   - 可选工具列表
   - 安装命令
   
4. **时间规划**
   - 本周任务
   - 下周任务
   - 月度计划

**输出**: 快速开始指南(Markdown)

## 最佳实践

### 1. 渐进式提问

不要一次性问所有问题,而是:
- 先问核心定位问题
- 根据回答深入细节
- 确认理解后再进入下一阶段

### 2. 提供具体示例

每个建议都应包含:
- ✅ 好的示例
- ❌ 不好的示例
- 💡 改进建议

### 3. 量化指标

所有策略都应包含:
- 可衡量的目标
- 时间节点
- 成功标准

### 4. 可执行性

确保所有建议:
- 有明确的执行步骤
- 提供必要的工具和资源
- 包含时间和成本估算

### 5. 灵活适配

根据用户反馈调整:
- 如果用户预算有限,推荐低成本方案
- 如果用户追求质量,推荐专业方案
- 如果用户时间紧迫,推荐快速方案

## 脚本和工具

### 1. Favicon 生成脚本

**位置**: `scripts/generate-favicon.sh`

**功能**:
- 从 Logo 生成全平台 Favicon
- 自动压缩优化
- 生成配置文件(site.webmanifest, browserconfig.xml)

**使用**:
```bash
./scripts/generate-favicon.sh brand-assets/logo/logo-icon.png
```

### 2. 品牌配置验证脚本

**位置**: `scripts/validate-brand-config.py`

**功能**:
- 验证 brand-config.json 格式
- 检查必需字段
- 验证配色对比度(WCAG AA)

**使用**:
```bash
python scripts/validate-brand-config.py brand-config.json
```

### 3. 文档生成脚本

**位置**: `scripts/generate-docs.sh`

**功能**:
- 根据模板生成完整文档
- 自动填充品牌信息
- 生成目录结构

**使用**:
```bash
./scripts/generate-docs.sh --config brand-config.json --output ./output
```

## 故障排查

### 问题: 生成的文档不完整

**检查**:
- 是否提供了足够的产品信息?
- 是否明确了目标受众?
- 是否确认了品牌调性?

**解决**:
- 重新执行阶段 1 需求分析
- 补充缺失的信息

### 问题: Logo 设计方案不满意

**检查**:
- AI Prompt 是否足够具体?
- 是否提供了参考案例?
- 配色方案是否符合品牌调性?

**解决**:
- 调整 Prompt 关键词
- 提供更多视觉参考
- 尝试不同的设计风格

### 问题: 脚本执行失败

**检查**:
- 依赖是否已安装(ImageMagick, pngquant)?
- 文件路径是否正确?
- 文件权限是否正确?

**解决**:
```bash
# macOS
brew install imagemagick pngquant

# Ubuntu
sudo apt-get install imagemagick pngquant

# 添加执行权限
chmod +x scripts/*.sh
```

## 注意事项

1. **版权问题**: 确保 Logo 设计不侵犯他人版权
2. **域名可用性**: 品牌名称确定前检查域名是否可注册
3. **配色对比度**: 确保符合 WCAG AA 标准(对比度 > 4.5:1)
4. **文件大小**: Logo 和 Favicon 应优化到合理大小
5. **多平台测试**: 在不同设备和浏览器测试视觉效果
