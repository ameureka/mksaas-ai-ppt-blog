# 关于图片的链接与加载机制（当前状态）

## 现状
- frontmatter `image` 统一为 `/images/blog/{category}/{slug}-cover.jpg`（英文 slug），文件实存于 `public/images/blog/{category}/`，封面均 <200KB。
- 列表卡片与详情页使用 `next/image` 默认 loader 加载本地静态资源，无远程存储/自定义 loader。
- 构建通过（`pnpm content`/`pnpm build`），缺图为 0。

## 分类化存储
- 分类目录：business / education / general / marketing / paid-search / proposal / report / year-end。
- 文件命名：`{slug}-cover.jpg`。

## Cloudflare 相关
- 当前未使用 Cloudflare Images/R2 或自定义 loader；图片随站点静态发布。
- 如部署在 Cloudflare（Pages/Workers）或前置 Cloudflare CDN，静态 `/public/images/...` 会自动缓存，提升命中与体验。
- 若需更强外链/存储：可将图片上传到 R2/Cloudflare Images，并在 `next.config.ts` 配置 remotePatterns/custom loader，frontmatter 写远程 URL（本仓暂未实现）。

## 建议
- 继续保持本地静态路径，结合 CDN 缓存即可；如要改为远程存储，再统一调整 frontmatter 路径与 loader 配置。
